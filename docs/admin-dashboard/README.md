## Overview

The admin dashboard is a web-based interface for managing all aspects of the Tunnel Server. It provides:

- **User Management**: Create, edit, and delete users
- **Token Management**: View and regenerate tunnel tokens
- **Tunnel Monitoring**: See all active tunnels
- **Activity Logs**: Audit trail of all actions
- **Statistics**: Real-time system metrics

### Dashboard URL

| Environment | URL |
|-------------|-----|
| Development | http://localhost:8000 |
| Production | http://your-server:8000 |
| With SSL | https://admin.yourdomain.com |

### Regenerating a Token

1. Click **"Token"** on the user
2. Click **"Regenerate Token"**
3. Confirm the action
4. Copy and share the new token

**Warning**: The old token stops working immediately.

### Enabling/Disabling Users

1. Find user in table
2. Click **"Disable"** or **"Enable"** button
3. Status changes immediately

**Disabled users:**
- Cannot log into dashboard
- Existing tunnels stop working
- Account preserved (can be re-enabled)

### Deleting Users

1. Click **"Delete"** button
2. Confirm in dialog
3. User and all their tunnels are deleted

**Warning**: This action cannot be undone.

## Tunnel Management

### Understanding Tunnel Status

Tunnels are managed by clients, not the admin dashboard. The dashboard provides monitoring:

| What Admin Can Do | What Admin Cannot Do |
|-------------------|----------------------|
| View all tunnels | Create tunnels |
| See connection status | Start/stop tunnels |
| See tunnel details | Delete tunnels |
| Monitor user quota | Configure tunnel settings |

### Tunnel Types

| Type | Description | Access Format |
|------|-------------|---------------|
| HTTP | Web traffic | subdomain.yourdomain.com |
| HTTPS | Secure web traffic | subdomain.yourdomain.com:443 |
| TCP | Raw TCP connections | yourdomain.com:port |

### Monitoring Tunnels

The Tunnels tab shows:

1. **Who owns the tunnel** - User email
2. **Tunnel configuration** - Name, type, subdomain/port
3. **Connection status** - Connected or Offline
4. **Last activity** - When tunnel was last used

## Activity Monitoring

### Activity Log Purpose

The activity log provides:

1. **Security Auditing** - Track who did what
2. **Troubleshooting** - Diagnose issues
3. **Compliance** - Record of all actions

### Log Retention

By default, logs are kept indefinitely. For cleanup:

```sql
-- Delete logs older than 30 days
DELETE FROM activity_logs
WHERE created_at < datetime('now', '-30 days');
```

### Searching Activity

The dashboard displays the most recent 50 activities. For searching:

```bash
# Use database directly
sqlite3 tunnel.db "SELECT * FROM activity_logs WHERE action = 'login' ORDER BY created_at DESC LIMIT 100;"
```

## Dashboard Features

### UI Theme

The dashboard uses a dark theme with these colors:

| Element | Color |
|---------|-------|

### Keyboard Shortcuts

Currently, no keyboard shortcuts are implemented. Navigation is mouse/touch-based.

### Mobile Responsiveness

The dashboard is responsive and works on:
- Desktop browsers
- Tablets
- Mobile devices (limited functionality)

### Auto-Refresh

The Overview tab auto-refreshes statistics every 10 seconds:

```javascript
setInterval(() => {
    if (document.getElementById('overviewTab').classList.contains('active')) {
        loadStats();
    }
}, 10000);
```

## Common Tasks

### Daily Operations

1. **Check Overview** - Verify active users and tunnels
2. **Review Activity** - Look for suspicious activity
3. **Monitor Resources** - Ensure tunnels within limits

### Weekly Operations

1. **User Audit** - Review active users
2. **Clean Up** - Disable unused accounts
3. **Token Review** - Regenerate tokens if needed

### Troubleshooting from Dashboard

| Symptom | Action |
|---------|--------|
| User can't connect | Check if account is Active |
| Too many tunnels | Increase max_tunnels or delete old tunnels |
| Suspicious activity | Check Activity tab, disable user if needed |
| Token compromised | Regenerate token immediately |
